This guide gives you the fastest path from install to your first successful publish, subscribe, search, and download.

## Before you start

- Install SCP2P from the [Installation guide](/guides/installation)
- Keep one bootstrap peer address ready (`ip:port`) if you are not on the same LAN as another node
- Make sure local firewall rules allow SCP2P traffic

## Path 1: desktop app (recommended)

<Steps>
  <Step title="Open Settings and save config">
    Set at least:
    - **State database path** (default is fine)
    - **Bind TCP** (default `0.0.0.0:7001` is fine)
    - **Bootstrap peers** (recommended when not LAN-only)

    Click **Save Config**.
  </Step>
  <Step title="Start the node">
    Go to **Dashboard** and click **Start Node**.

    Expected result:
    - Node status becomes **Active**
    - You see the TCP bind address in the status card
  </Step>
  <Step title="Publish your first share">
    Open **My Shares** and click **New Share**.

    Choose **Files** or **Folder**, set title + visibility, then click **Publish**.
  </Step>
  <Step title="Copy a share link">
    In **My Shares**, click **Copy Link** for your share and send it to another user.
  </Step>
  <Step title="Subscribe and sync">
    On another node, open **Discover**, click **+**, paste the link, then click **Sync**.
  </Step>
  <Step title="Search and download">
    Open **Search**, find a file, and click **Download**.
  </Step>
</Steps>

## Path 2: CLI interactive shell

SCP2P CLI is interactive. The executable command is `scp2p`.

<Steps>
  <Step title="Start the shell">
    ```bash
    scp2p --db scp2p.db --port 7001 --bootstrap 203.0.113.10:7001 --log-level info
    ```

    The shell starts a local node and opens a menu.
  </Step>
  <Step title="Publish content">
    Choose **Publish files** or **Publish folder**.

    Set:
    - Title
    - Visibility (`private` or `public`)
    - File/folder path
  </Step>
  <Step title="Subscribe">
    Choose **Subscriptions** -> **Subscribe to a new share** and paste:
    - Share ID
    - Optional share public key
  </Step>
  <Step title="Sync, search, and download">
    Choose **Sync now**, then **Search**, then **Download by content ID**.

    You can also use **Download share** to fetch multiple files at once.
  </Step>
</Steps>

## Verify success

You are done when all of these are true:

- Dashboard/CLI status shows a running node
- At least one subscription appears
- Search returns results
- A downloaded file is present on disk

## Build from source (optional)

If you want to run the latest source:

```bash
git clone https://github.com/techartdev/scp2p.git
cd scp2p
cargo install --path crates/scp2p-cli
```

Then run:

```bash
scp2p --db scp2p.db --port 7001
```

## Troubleshooting

- `No peers configured`: add bootstrap peers in app **Settings** or via CLI `--bootstrap`.
- `Node won’t start`: check if port `7001` is already in use.
- `Subscribed but no files`: run **Sync** and retry browse/search.
- `Download fails`: verify at least one peer with content is online.

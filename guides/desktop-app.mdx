The desktop app is the best way to use SCP2P if you want a visual workflow for publishing, subscribing, and downloading.

## Install

Download a release build from [GitHub Releases](https://github.com/techartdev/scp2p/releases).

| OS | Installer Type |
|---|---|
| **Windows** | `.msi` or `.exe` |
| **macOS** | `.dmg` or `.app.tar.gz` |
| **Linux** | `.AppImage` or `.deb` |

Install and launch the app.

## First run checklist

Before you do anything else:

1. Open **Settings**
2. Set **State database path**
3. Set **Bind TCP** (default `0.0.0.0:7001` is usually fine)
4. Add at least one **Bootstrap Peer** if you are not LAN-only
5. Click **Save Config**
6. Go to **Dashboard** and click **Start Node**

Expected result:
- Node status changes to **Active**
- Peers appear over time in Dashboard or Discover

## Sidebar pages

### Dashboard

- Start or stop the local node
- Check peer/subscription/community counts
- See bind addresses, DB path, and warnings
- Use quick links to jump to Discover, My Shares, and Settings

### Discover

- View discovered and subscribed shares
- Subscribe using share ID or `scp2p://s/...` link
- Run **Sync** to fetch new manifests
- Browse share file trees
- Select files/folders and queue downloads

### Search

- Search across subscribed catalogs only
- View relevance score and snippet
- Save a result to any local path

### My Shares

- Publish files or folders
- Choose `public` or `private` visibility
- Copy share links for other users
- Export share secret key when needed
- Republish to create a new share revision
- Toggle visibility or delete a share

### Communities

- Join or leave communities
- Paste community share links to auto-fill join fields
- Browse community participants and community public shares

### Settings

- Load and save `scp2p-desktop-config.cbor`
- Configure:
  - DB path
  - TCP/QUIC bind values
  - Bootstrap peers
  - Auto-start on app launch

## Recommended daily workflow

<Steps>
  <Step title="Start node">
    Open **Dashboard** and click **Start Node**.
  </Step>
  <Step title="Sync content">
    Open **Discover** and click **Sync**.
  </Step>
  <Step title="Publish or subscribe">
    Publish in **My Shares** or subscribe in **Discover**.
  </Step>
  <Step title="Search and download">
    Use **Search** for content lookup and **Discover** for tree-based multi-file download.
  </Step>
</Steps>

## Troubleshooting

- `NodeRequired overlay keeps appearing`: node is stopped; return to Dashboard and start it.
- `No peers found`: add reachable bootstrap peers in Settings and retry Sync.
- `Share subscribed but empty`: run Sync and reopen the share in Discover.
- `Downloads stall`: keep publisher or another provider online during transfer.

## Developer mode (build from source)

If you are contributing to the app:

```bash
git clone https://github.com/techartdev/scp2p.git
cd scp2p/app
npm install
npm run tauri dev
```

The `crates/scp2p-core` crate exposes SCP2P runtime, protocol, and transport primitives.
For application integration, you primarily use `Node` and `NodeHandle`.

## Start a node

Start with `NodeConfig`, then call `Node::start` or `Node::start_with_store`.

```rust
use scp2p_core::{Node, NodeConfig};

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let config = NodeConfig::default();
    let node = Node::start(config).await?;
    let _node_key = node.ensure_node_identity().await?;
    Ok(())
}
```

`NodeConfig` controls bind addresses, capabilities, bootstrap peers, subscription limits, publisher-key auto-protection, and community strict mode.

## Main exported types

- Runtime and lifecycle: `Node`, `NodeHandle`, `NodeConfig`
- IDs and manifests: `NodeId`, `ShareId`, `ContentId`, `ManifestId`, `ManifestV1`, `ShareHead`, `ShareKeypair`, `ShareVisibility`
- Peers and transport: `PeerAddr`, `TransportProtocol`, `Capabilities`, `FetchPolicy`, `RequestTransport`, `PeerConnector`
- Search: `SearchQuery`, `SearchPageQuery`, `SearchPage`, `SearchResult`, `SearchTrustFilter`
- Communities and policy: `CommunityMembershipToken`, `SubscriptionTrustLevel`, `BlocklistRules`
- Relay: `RelayLimits`, `RelayAnnouncement`, `ActiveRelaySlot`, `AbuseLimits`
- Persistence: `Store`, `MemoryStore`, `SqliteStore`, `PersistedState` and related persisted record types

## Membership tokens

`CommunityMembershipToken` provides:
- `CommunityMembershipToken::issue(...)`
- `CommunityMembershipToken::verify(...)`

Use this when you want cryptographically provable community membership between nodes.

## Note on signatures

Some APIs are generic over `RequestTransport` or `PeerConnector`, and some functions have many arguments for explicit protocol control. For exact signatures, use the source in `crates/scp2p-core/src/api/*.rs`.
